Never

  • Stretch, skew, rotate, or flip
  • Add gradients, bevels, glows, or drop shadows
  • Outline the wordmark or change its typeface
  • Place in a circle or badge that isn't the app icon

Rules

  1. Shell is never a status colour. Brand orange means Clawman — action, identity, brand. It never means "warning." Buoy amber sits close in hue, which is exactly why the status scale is defined independently.
  2. One Shell per view. One primary button, or one active state, or one accent. If everything's orange, nothing is.
  3. Dark is the default. Clawman lives in an editor. Design dark first, derive light.
  4. Status colour is never the only signal. Always pair with an icon and a text label.
  5. Never tint large surfaces with Shell. No orange backgrounds or cards. Fills use shell-100 at most.
  6. Borders are one step, not two. deep-600 on dark, sand-300 on light. Don't invent intermediate greys.

Verified contrast

PairingRatioLevel

shell-500 on light sits at 3.5:1 — large text and graphics only. Use shell-600 or shell-700 for body copy.

Rules

  1. Never set body copy in Bricolage. Nothing under 24 px.
  2. Never set headings in JetBrains Mono. Monospace means one thing; keep it meaning only that.
  3. Two weights per screen. Regular plus one emphasis weight. Weight carries hierarchy before size does.
  4. Measure 60–75 characters for body text. Marketing copy sits at the lower end.
  5. Sentence case everywhere. No all-caps except Micro badges, at .06em tracking.
  6. Never fake a weight. No synthetic bold or italic — load the real cut.

All three families are open source. A developer tool should ship a brand anyone can rebuild without a licence.

Tone of voice

Meticulous, not fussy. Warm, not cute. Dry, not snarky. Direct, not curt. The humour is understatement, and it's seasoning — a tool that's constantly winking is a tool you stop trusting.

  1. Lead with the noun. Say what happened to what.
  2. Short sentences. Concrete words. If a sentence needs a comma to survive, it needs a full stop instead.
  3. Never inflate. Banned: seamlessly, effortlessly, supercharge, revolutionise, leverage, unlock, game-changing, next-generation, powerful, magic.
  4. "You" for the user, "Clawman" for the product. "We" only in the changelog.
  5. One joke per screen, maximum.

Conventions

NameClawman capitalised in prose, lowercase in the wordmark. Never ClawMan, clawMan, or Claw Man.
PunctuationNo exclamation marks in product UI. One per release in the changelog, if earned. Oxford comma: yes.
EmojiNone in product UI. Sparingly in README and release notes.
CaseSentence case for every heading, button, and label. Never Title Case.
PathsEvery file name in monospace, always.

Layout, shape and motion

Spacing4 px base. Use 4, 8, 12, 16, 24, 32, 48, 64, 96. Nothing between.
Radii4px inputs and badges · 8px cards and buttons · 12px panels and modals · 999px pills only.
Borders1 px, always. Emphasis comes from surface elevation and colour, not thicker strokes.
ElevationOn dark, elevate with surface colour (deep-900 → 800 → 700), not shadow. On light, one shadow only.
Motion120 ms state, 200 ms entrance, cubic-bezier(.2,0,0,1). Nothing bounces — precise things don't wobble.
Icons1.5 px stroke, round caps and joins, 24 px grid, matched to the mark. Lucide recommended.

Before you ship

  1. Exactly one Shell accent in this view?
  2. Every status colour paired with an icon and a label?
  3. All display type above 24 px, all body type in Inter?
  4. All file paths in monospace?
  5. Sentence case throughout — no Title Case, no all-caps?
  6. Zero exclamation marks, zero banned words?
  7. At most one joke on screen?
  8. Does the copy name the actual thing — the file, the count, the date?
  9. Mark has full clearspace, in an approved colourway?
  10. Every text pairing clears 4.5:1?
