The problem nobody watches

Context files start clean. One CLAUDE.md, written carefully on a good afternoon. Six weeks later there are eleven of them, two contradict each other about the test command, and one describes a directory that was deleted in March.

Nobody has read any of them since the day they were written. Except Claude. Every single time.

01

Everything, in one place

Finds CLAUDE.md, .claude/ directories, skills, and memory files anywhere in the workspace — including the ones you forgot you wrote.

02

Stale files, flagged

When a context file stops changing while the code it describes keeps moving, Clawman says so. With a date, not a vibe.

03

Contradictions, surfaced

Two files giving Claude opposite instructions is worse than no instructions at all. Clawman shows both excerpts side by side.

04

Dead references, caught

Context files love to describe directories that no longer exist. Clawman checks every path they mention.

05

Edit in place

Merge two files, archive one, or jump straight to the line that's wrong. No leaving the editor.

06

Nothing leaves your machine

Clawman reads files on disk. No telemetry, no uploads, no account. It's a filing clerk, not a service.

Three steps, then it's just there

Install the extension

From the VS Code Marketplace, or ext install clawman.clawman in the command palette.

Open the Clawman panel

It appears in the activity bar. First scan takes about a second on a normal repo.

Read what he found

Four states: fresh, stale, conflict, archived. Sorted worst-first, because that's the list you actually want.

Built in the open

Every change lands in the dev log first — what shipped, what broke, and what Clawman noticed along the way.
